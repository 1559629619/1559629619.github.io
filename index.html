跨域问题随笔

在项目中遇到了no access-control-allow-origin报错，了解到是跨域导致的问题。

什么是跨域

出于浏览器的同源策略限制。同源策略（Sameoriginpolicy）是一种约定，它是浏览器最核心也最基本的安全功能，如果缺少了同源策略，则浏览器的正常功能可能都会受到影响。可以说Web是构建在同源策略基础之上的，浏览器只是针对同源策略的一种实现。同源策略会阻止一个域的javascript脚本和另外一个域的内容进行交互。所谓同源（即指在同一个域）就是两个页面具有相同的协议（protocol），主机（host）和端口号（port）
跨域问题出于浏览器的同源策略限制，他是为了保证用户的安全，防止用户信息被窃取，比如ajax请求只能发给同源的网址，否则就会报错，比如我在项目中前后端用的端口不同，如果不进行配置则会报跨域错误，某宝也不能随意成功请求某讯的后台获取数据。

我们在开发的时候，可能暂时不需要这些安全防护，可以进行配置跳过跨域防护。

问题解决

  1. 返回新的CorsFilter（全局跨域）
    在任意配置类，返回一个新的CorsFilter Bean，并添加映射路径和具体的CORS配置信息。
  2. 重写WebMvcConfigurer（全局跨域）
    在任意配置类，返回一个新的WebMvcConfigurer Bean，并重写其提供的跨域请求处理的接口，目的是添加映射路径和具体的CORS配置信息。

    举一个简单的例子，对于一个long型变量的赋值和取值操作而言，在多线程场景下，线程A调用set（100），线程B调用get（），在某些场景下，返回值可能不是100。这是因为JVM的规范并没有要求64位的long或者double的写入是原子的。在32位的机器上，一个64位变量的写入可能被拆分成两个32位的写操作来执行。这样一来，读取的线程就可能读到“一半的值”。解决办法也很简单，在long前面加上volatile关键字。

HashMap

1.概述

​ HashMap底层基于散列算法实现。HashMap 允许 null 键和 null 值，在计算哈键的哈希值时，null 键哈希值为 0。HashMap 并不保证键值对的顺序，这意味着在进行某些操作后，键值对的顺序可能会发生变化。HashMap 是非线程安全类，在多线程环境下可能会存在问题。

2.原理

​ HashMap采用了拉链式的散列算法，数据结构采用数组+链表（JDK8之后为数组+链表+红黑树）。进行增删查操作时，先定位到元素的所在桶的位置，之后再从链表中定位该元素。

TCP/IP4层结构

Posted on 2021-09-08

TCP/IP4层结构

  1. 应用层

    向用户提供应用服务，包括文件传输协议FTP、域名系统DNS、超文本传输协议HTTP。

  2. 传输层

    上对应用层，提供两台计算机间的数据传输服务，下对网络层，将TCP/UDP报文封装到I报文里。

  3. 网络层

    网络层用来处理网络上流动的数据包，他规定了怎样到达对方计算机（路由选择），把数据包传送给对方。

  4. 数据链路层

    主要用来处理物理部分，包括网络适配器、操作系统、设备驱动等物理可见部分，并进行了差错控制。

Mysql

聚合函数

  • COUNT(*)计算表中总的行数，不管某列是否有数值或者为空值。

    COUNT(字段名)计算指定列下总的行数，计算时将忽略空值的行。

  • AVG()函数通过计算返回的行数和每一行数据的和，求得指定列数据的平均值。AVG()函数()

    AVG()函数可以与GROUP BY一起使用，来计算每个分组的平均值。

  • SUM()是一个求总和的函数，返回指定列值的总和。

    SUM()可以与GROUP BY一起使用，来计算每个分组的总和。

  • MAX()返回指定列中的最大值。

    MAX()也可以和GROUP BY关键字一起使用，求每个分组中的最大值。

    MAX()函数不仅适用于查找数值类型，也可应用于字符类型。

  • MIN()返回查询列中的最小值。

    MIN()也可以和GROUP BY关键字一起使用，求出每个分组中的最小值。

    MIN()函数与MAX()函数类似，不仅适用于查找数值类型，也可应用于字符类型。

缓存穿透 缓存击穿 缓存雪崩

Posted on 2021-08-25 Edited on 2021-08-31

缓存穿透 缓存击穿 缓存雪崩

  • 缓存穿透

    • 缓存穿透是指客户端请求根本不存在的key，Redis中查询不到，使得请求直达数据库，导致数据库负载过大甚至宕机。一般是由于业务层误删数据或者是有人恶意攻击。

    • 解决方案：

      1. 缓存空值，当Redis未命中时缓存该key的空值，当下次访问的时候缓存直接返回空值。此方法会导致冗余，可能会降低性能。
      2. 布隆过滤器，把数据存入布隆过滤器，在访问缓存之前过滤器会进行拦截，根据算法判断请求的数据是否存在，如果不存在则直接返回空值。

SpringMVC

Posted on 2021-08-24

SpringMVC

  • DispatcherServlet(重中之重)
  • HandlerMapping
  • HandlerAdapter(主要干活的就是他，其他的主要是指挥调配，调用controller)
  • ViewResolver

流程

  1. 前端发送请求给DispatcherServlet

  2. DispatcherServlet向HandlerMapping询问谁能处理这些请求

  3. HandleMapping查询到相关组件后返回一个HandlerExecutionChain给DispatcherServlet（因为可能需要多个组件才能完成相应的工作）

  4. DispatcherServlet根据返回的组件获取HandlerAdapter，调用它去执行Controller

  5. 执行完毕后适配器返回结果ModelAndView给DispatchServlet

  6. DispatcherServlet拿着ModelAndView交给ViewResolver进行视图解析

  7. ViewResolver解析完将结果View返回给DispatcherServlet

  8. DispatcherServlet对View进行渲染

  9. 最后DispatcherServlet把结果返回给用户（响应）

     *如果方法加上了@ResponseBody注解 则返回Json对象 此时由HandlerAdapter直接返回 不需要经由ViewResolver
